The Boring Project——驾校管理系统

8月 11 2019 Project 3 分钟 读完 (约 427 字)

一、关于项目由来

​ 赋闲蜗居，空余时间颇多，碰巧好友驾校公司需要管理系统对学员、教员、车辆等档案进行统一管理，方便无纸化办公。

二、功能需求

三、架构

1、前端

框架选择：

  • Vue.js

    前端路由使用动态路由，根据角色

  • ElementUI

2、后端服务

框架选择：

  • Node.js

  • express：选择express 作为web框架，提供RESTful API服务。

    选择express的原因主要是用过，社区支持相对比较多，总体来说Express大全，基本满足需求，koa 结构精简，要什么找对应中间件。

    egg.js、think.js这些不考虑。

  • mongodb：数据库选择MongoDB。

    • noSql、速度快、BSON（类JSON）的文档存储模式，文档结构修改方便简单。目前项目主要功能还是以档案存储为主，类JSON的写法比较熟悉合适。
    • CRUD 相对比较简单。
    • 对于一对多、多对多的关系可以用内嵌文档、或者关系引用的方式加快查询速度。
    • 对于联表查询、计算、统计分组等可以使用db.aggregation()

3、服务器部署

  • Docker

    Docker-compose 编排容器搞定一切～

    node server 在容器中可以使用pm2 进程守护、监控服务，日志输出可以从容器映射到宿主主机。
